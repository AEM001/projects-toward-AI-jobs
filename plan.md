# AI 应用工程师技能图谱与成长路线

本文档旨在明确“AI 应用工程师”方向的核心技能树与知识点。目标是构建 **“AI 应用 + 工程基础 + 系统部署”** 的复合型能力，不仅具备独立开发能力，更具备解决实际业务问题的能力。

---

## 1. 核心方向：AI 应用开发 (AI Application Engineering)
*这是主业核心，决定了你能否胜任当前 AI 落地需求。*

### 基础大模型能力 (LLM Foundations)
- **API 调用**: 熟练集成主流模型 (OpenAI, DeepSeek, Claude, Llama 等)。
- **Prompt Engineering**: 结构化提示词设计、思维链 (CoT) 优化。
- **Embeddings**: 文本向量化原理与应用。
- **Function Calling**: 让大模型调用外部工具（API、数据库、搜索）的能力。

### RAG (检索增强生成)
- **向量数据库**: Milvus, Chroma, Pinecone 等的使用与维护。
- **数据处理 Pipeline**: 
  - 多格式文档解析 (PDF, Markdown, Word)。
  - 分块策略 (Chunking Strategies)。
- **检索优化**: 
  - 混合检索 (关键词 + 向量)。
  - Rerankers (重排序模型) 的使用。
- **架构设计**: 索引构建 -> 检索召回 -> 上下文组装 -> 生成。

### Agent (智能体) & Workflow
- **Agent 架构**: 理解 ReAct, Plan-and-Solve 等模式。
- **工作流编排**: 
  - 任务拆解 (Task Decomposition)。
  - 长期记忆机制 (Memory)。
- **多 Agent 协作**: 角色定义与任务分发。
- **工具集成**: 能够封装邮件、日历、数据库、搜索等为 Agent 可用工具。

### 多模态 (Multi-modal)
- **视觉**: OCR (文档/发票识别), CLIP, Vision-LLM (图像理解)。
- **音频**: ASR (语音转文字), TTS (文字转语音), 会议纪要自动生成。

---

## 2. 工程基石：后端与软件工程 (Backend & Software Engineering)
*这是支撑 AI 应用落地的基础，决定了系统的稳定性与可维护性。*

### Python 工程化
- **项目结构**: 规范的目录结构 (MVC/DDD), 模块化设计。
- **环境与依赖**: venv, poetry, requirements.txt 管理。
- **代码质量**: Logging (日志体系), Exception Handling (异常处理), Type Hinting。
- **异步编程**: `async/await` 并发处理，提高 I/O 密集型任务效率。

### Web 开发框架
- **FastAPI** (首选):
  - 路由与中间件。
  - Pydantic 数据验证与 Schema 定义。
  - 依赖注入 (Dependency Injection)。
- **API 设计**: RESTful 规范, HTTP 状态码, JSON 数据交互。
- **安全鉴权**: JWT (JSON Web Token), RBAC (基于角色的权限控制)。

### 数据库与中间件
- **关系型数据库**: PostgreSQL (推荐，对向量插件支持好) 或 SQLite。
- **缓存系统**: Redis (缓存热点数据, Session 管理, 简单的消息队列)。
- **ORM**: SQLAlchemy 或 Tortoise-ORM。

### 计算机基础
- **数据结构**: 列表、字典、树、图的基本操作。
- **设计模式**: 单例模式、工厂模式、策略模式等在工程中的应用。
- **版本控制**: Git 高级操作 (Branching, Merging, PR流程)。

---

## 3. 运维与部署：系统工程 (DevOps & Deployment)
*这是加分项，决定了你能否独立交付完整的商业系统，实现“一个人就是一支队伍”。*

### 容器化技术
- **Docker**: 
  - 编写高效的 Dockerfile。
  - 镜像构建与优化。
  - Docker Compose 多服务编排 (Frontend + Backend + DB + VectorDB)。

### 服务器与部署
- **Web 服务器**: Nginx 配置 (反向代理, 负载均衡, HTTPS)。
- **云服务基础**: 阿里云/AWS 基础组件 (ECS/EC2, OSS/S3)。
- **Linux 基础**: 常用 Shell 命令, 脚本编写, 权限管理。

### CI/CD & 监控
- **自动化**: GitHub Actions (自动测试, 自动构建 Docker 镜像)。
- **可观测性**: 基础的服务监控与日志聚合。

---

## 4. 软技能：业务与产品 (Business & Product)
*这是不可替代性的来源，决定了你能否将技术转化为价值。*

- **需求翻译**: 将模糊的业务需求转化为具体的 AI 技术方案 (Token 成本估算, 延迟评估, 可行性分析)。
- **产品思维**: 理解业务流程 (SOP)，寻找 AI 提效的切入点 (Automation)。
- **快速原型**: 能够使用 Streamlit, Gradio 或简单前端快速搭建 Demo 进行验证。
- **沟通能力**: 清晰阐述技术方案的商业价值 (降本增效)。

## 7. 学习建议

### 时间投入
- **每周 15-20 小时**（平均每天 2-3 小时）
- 周末集中攻坚，工作日巩固学习

### 学习方法
1. **先跑通 Demo**：不求完美，先让项目运行起来
2. **边做边学**：遇到问题现查文档，不要提前学完所有理论
3. **强制输出**：每周必须有可展示的成果，拒绝"学了但没做"
4. **代码开源**：所有项目上传 GitHub，养成良好习惯
5. **写技术笔记**：每个项目写一篇总结（遇到的坑、解决方案）

### 资源推荐
- **文档优先**：官方文档 > 博客 > 视频教程
- **AI 辅助**：使用 ChatGPT/Claude 解决具体问题，但不要完全依赖
- **社区参与**：加入相关技术社区，观察别人的项目实现

### 调整策略
- 如果某周项目太难，可以简化需求但必须完成
- 如果某周项目太简单，可以增加额外功能
- 保持节奏比完美更重要，**持续输出是核心**
